# 📊 Olist E-commerce Data Analysis

This project focuses on analyzing a real-world Brazilian e-commerce dataset (Olist) to understand business performance, customer behavior, and delivery insights.

## 🚀 Project Overview

In this project, I worked on the complete data analysis pipeline:

* Data cleaning and preprocessing
* Merging multiple datasets
* Feature engineering
* Exploratory data analysis (EDA)
* Extracting business insights

The goal was to move beyond basic analysis and understand **why certain trends occur** in the business.

---

## 🛠️ Tools & Technologies

* Python
* Pandas
* Jupyter Notebook / VS Code
* Power BI (for dashboard - upcoming)

---

## 📂 Dataset Information

The dataset used is the **Olist Brazilian E-commerce Dataset**.

⚠️ Due to file size limitations, the dataset is not included in this repository.
You can download it from Kaggle.

---

## 🔍 Key Analysis Performed

### 1. Revenue Analysis

* Identified top-performing cities and states
* Analyzed revenue by payment types

### 2. Product Category Insights

* Found high and low revenue categories
* Compared number of orders and seller distribution

### 3. Delivery Performance

* Calculated delivery time
* Categorized delivery speed
* Analyzed impact of delivery on customer ratings

### 4. Customer Behavior

* Identified repeat customers
* Analyzed order frequency

### 5. Seller Performance

* Revenue generated by sellers
* Number of orders handled

---

## ⚙️ Feature Engineering

Created new columns like:

* `delivery_time`
* `delivery_type`
* `delivery_delay`
* `is_delayed`
* `is_repeat_customer`
* `order_month`

---

## 📈 Key Insights

* High revenue categories like *health_beauty* have more orders and wider seller distribution
* Low revenue categories have limited seller presence and fewer orders
* Faster delivery is generally linked with better customer ratings
